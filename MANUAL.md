## 1. このシステムでできること

スマートフォンで歩き方を計測して、**「歩行年齢」** を診断するシステムです。

```
スマートフォンで歩く
       ↓
データを自動解析
       ↓
歩行年齢・スコアがわかる
       ↓
A4 レポートを PDF で出力・共有
```

**診断できる内容**

| 項目 | 説明 |
|------|------|
| 歩行年齢 | 「何歳相当の歩き方か」を推定 |
| 総合スコア | 0〜100 点で評価 |
| 歩くスピード | m/s（毎秒何メートル進むか） |
| 歩幅の広さ | 1 歩の長さ（m） |
| 歩くテンポ | 1 分間の歩数（歩/分） |
| リズムの安定性 | 一定のリズムで歩けているか |
| 左右対称性 | 左右の歩き方のバランス |

**画面右上のボタン**

| ボタン | 機能 |
|--------|------|
| **地域在住向け** | ON にすると Page 2（地域在住者・介護予防向け評価レポート）が表示される。デフォルトは OFF |
| **臨床向け** | ON にすると Page 3（医療・介護専門職向け詳細データ）が表示される。デフォルトは OFF |
| **シューズレコメンド** | ON にすると Page 4（シューズレコメンド）が表示される。デフォルトは OFF |
| **詳細アドバイス** | ON にすると Page 1.5（推奨介入・運動アドバイス）が表示される。デフォルトは OFF |
| **ガイド** | 計測ガイド（3 ステップ）を表示する |
| **音声** | 音声フィードバックの ON/OFF を切り替える |
| **印刷** | ブラウザの印刷ダイアログを開く |
| **共有** | PDF を生成して OS 共有メニューを開く |
| **PDF出力** | PDF を生成してダウンロードする。同時に計測結果が履歴データベースに自動保存される |

## 2. 必要なものを用意する

### 2-1. 機器・アプリ

| 必要なもの | 詳細 |
|-----------|------|
| **スマートフォン** | iPhone または Android（センサー内蔵のもの） |
| **phyphox アプリ** | 無料（CSV取込モード・phyphox WiFiリモートモード使用時のみ必要。直接計測モードでは不要） |
| **PC またはタブレット** | Chrome / Safari / Edge など最新ブラウザ推奨 |

### 2-2. 計測環境

- **歩く場所：** 直線で **推奨 10m（5m・20m・任意距離にも対応）** 歩けるスペース
- **床の状態：** 平坦な床（屋内廊下・体育館など）
- **靴：** 普段履いている靴でOK
- **補助具：** 杖・歩行器を使っている方はそのまま計測可能

> **ポイント：** 10 m のラインをテープなどで床に貼っておくと計測が楽になります。

## 3. phyphox アプリのセットアップ

phyphox（フィフォックス）は、スマートフォンの加速度センサーを使って計測するアプリです。**無料**で使えます。

### 3-1. インストール

| 端末 | 入手先 |
|------|--------|
| iPhone (iOS) | App Store で「phyphox」を検索 |
| Android | Google Play で「phyphox」を検索 |

> 開発元：RWTH Aachen University（ドイツのアーヘン工科大学）

### 3-2. 計測メニューの設定

1. phyphox を起動する
2. 画面下の **「+」ボタン** をタップ
3. **「Acceleration (without g)」**（重力を除いた加速度）を選ぶ

   > 日本語表示の場合は **「加速度（重力補正）」** または **「Linear Acceleration」**

4. 新しい実験が作成される

### 3-3. WiFi リモート機能の設定（任意・便利機能）

計測中にスマートフォンを操作しなくて済む方法です。スマートフォンと PC が**同じ WiFi**（院内 LAN など）に接続されている場合に使えます。

1. phyphox の実験画面右上の **「⋮」（縦三点）→「Remote Access」** をタップ
2. **「Access Address」** もしくはIPアドレス（例：`http://192.168.1.5`）が表示される
3. パソコンの画面で、そのIPアドレス（例：`192.168.1.5`）を入力し、「取得」ボタンを押します。
4. 表示された **IP アドレス** をメモしておく

> ⚠️ **WiFi リモート機能はローカルLAN環境（同一ネットワーク内）専用です。Netlify 公開 URL および GAS デプロイ URL からはご利用いただけません。** CSVファイルのエクスポート＆アップロード（方法A）をご利用ください。
>
> **ローカル起動（`file://` または `http://localhost` など）の HTTPS 設定：**
> ブラウザの Mixed Content 制限のため、phyphox で「**Use HTTPS**」を有効にし、PC ブラウザでそのアドレス（例：`https://192.168.1.5`）を開いて証明書を承認してから使用してください。

## 4. 歩行計測の手順

### 4-1. スマートフォンの装着位置

腰のベルトや**ズボンの腰部分**（腰の中央・背骨の真後ろ付近）に固定するのが最適です。

```
      [頭]
       │
      [肩]
       │
  ━━━━━━━━  ← 腰（ここにスマートフォンを固定）
       │
      [足]
```

**固定方法（どれか 1 つ）**
- ポケットに入れる（腰ポケット）
- ウエストポーチに入れて腰に巻く
- スマートフォンホルダーをベルトに装着
- スポーツアームバンドを腰に巻いて装着

> **注意：** 手に持って歩くと正確に計測できません。必ず腰に固定してください。

### 4-2. 計測手順（ステップバイステップ）

```
① 準備する（10m ライン）
   ↓
② phyphox で計測スタート
   ↓
③ 3〜5 秒 その場で静止（準備のため）
   ↓
④ 普通のペースで 10m 歩く
   ↓
⑤ 10m 歩いたら折り返す（または止まる）
   ↓
⑥ ①〜⑤ を 2〜3 往復繰り返す（計 20〜30 秒以上）
   ↓
⑦ phyphox で計測ストップ
   ↓
⑧ データをエクスポート（次ページ参照）
```

> **ポイント：** 「テスト中」と意識しすぎず、**いつも通りのペース**で歩いてください。

### 4-3. phyphox からデータをエクスポートする

**方法A：CSVファイルに保存する**

1. 計測停止後、phyphox 右上の **「⋮」→「Export Data」** をタップ
2. **「CSV（Comma Separated Values）」** を選択
3. メールや AirDrop などで PC に送る

**方法B：WiFi リモートでPCから直接取得する（ローカルLAN専用）**

> ⚠️ **Netlify・GAS 公開 URL からは利用できません。** ローカル起動（file:// または http://localhost）環境のみ対応。

前提：phyphox の「Remote Access」が有効であること（3-3 参照）。ローカル起動の場合は「Use HTTPS」設定とブラウザでの証明書承認も必要。

1. phyphox で計測開始・歩行・計測停止まで完了させる
2. 計測システムの左パネル「phyphox WiFi リモート取得」欄に IP アドレスを入力
3. 「取得」ボタンを押すと自動でデータが読み込まれる

### 4-4. 直接計測モード（phyphox なしで計測する方法）

スマートフォンのブラウザから直接加速度を計測するモードです。phyphox のインストールが不要です。

**必要条件**
- HTTPS 環境（Netlify 公開 URL または GAS デプロイ URL）
- Android Chrome または iOS Safari 13 以降

**手順**

1. **スマートフォンのブラウザ**で計測システムの URL を開く
2. 左パネルの計測データ取得セクションで **「📱 直接計測」タブ** を選択
3. **「計測を開始する」** ボタンをタップ
4. iOS の場合、センサーの使用許可を求めるダイアログが表示されるので **「許可」** をタップ
5. スマートフォンを**腰に固定**し、**3〜5 秒静止**してから歩き始める
6. 画面にリアルタイム波形・経過時間・サンプル数が表示される
7. **15 秒以上**歩くと「停止して分析」ボタンが有効になる
8. 歩行が終わったら **「停止して分析」** をタップ
9. 自動的にデータが解析され、レポートが生成される

> **注意：** ローカル環境（`file://`）では動作しません。HTTPS 環境で使用してください。

## 5. HTMLファイルの入手と起動

### 5-1. ファイルの入手

担当者から以下のいずれかの方法でファイルを受け取ります：

| 受け取り方法 | 手順 |
|------------|------|
| **ファイルを直接もらう** | `index.html` を PC の任意のフォルダに保存 |
| **GAS の URL を共有してもらう** | ブラウザで URL を開くだけ（インストール不要） |
| **Netlify URL を共有してもらう** | ブラウザで URL を開くだけ（インストール不要） |

### 5-2. HTML ファイルをローカルで開く

1. `index.html` ファイルをダブルクリック
2. ブラウザで自動的に開く

> **補足：** ローカル（自分の PC）で開いた場合、phyphox WiFi リモート取得と ASICS 商品画像の一部機能は動作しません。CSVファイルのアップロードは問題なく使えます。

## 6. Google Apps Script（GAS）でのセットアップ

GAS を使うと、URL をスタッフで共有するだけで複数人が使える環境を作れます。

### 6-1. Google アカウントの準備

Google アカウント（Gmail アカウント）が必要です。持っていない場合は [accounts.google.com](https://accounts.google.com) で作成してください。

### 6-2. Apps Script プロジェクトを作成する

1. ブラウザで [script.google.com](https://script.google.com) を開く
2. 左上の **「新しいプロジェクト」** をクリック
3. プロジェクト名を「歩行年齢テスト」などに変更（左上の「無題のプロジェクト」をクリックして入力）

### 6-3. Code.gs を設定する

1. 画面左に **「Code.gs」** が表示されている（デフォルトで作成済み）
2. `Code.gs` をクリックして開く
3. 最初から書かれているコードを **すべて削除**
4. プロジェクトの `Code.gs` ファイルの内容を**丸ごとコピー＆ペースト**
5. 上部の **「💾 保存」ボタン**（または Ctrl+S / Cmd+S）で保存

```
【Code.gs に貼り付ける内容のイメージ】

function doGet(e) { ... }         ← HTMLページを配信する関数
function fetchCsvData(url) { ... } ← シートデータを取得する関数
function fetchImageAsBase64(url) { ... } ← 画像を取得する関数
```

### 6-4. index.html を作成する

1. 画面左の **「＋」ボタン → 「HTML」** をクリック
2. ファイル名を **「index」**（拡張子なし）と入力して Enter
3. `index.html` ファイルが作成される
4. プロジェクトの **`index.html`** の内容を**丸ごとコピー＆ペースト**
5. 保存（Ctrl+S / Cmd+S）

### 6-5. ウェブアプリとしてデプロイする

1. 画面右上の **「デプロイ」→「新しいデプロイ」** をクリック
2. 「種類を選択」右の**歯車アイコン** をクリック → **「ウェブアプリ」** を選択
3. 以下のように設定する

   | 項目 | 設定値 |
   |------|--------|
   | 説明（任意） | 歩行年齢テスト v7 |
   | 次のユーザーとして実行 | **自分** |
   | アクセスできるユーザー | **全員** |

4. **「デプロイ」** ボタンをクリック
5. 初回は「アクセスを承認」の画面が出る → **「アクセスを承認」** をクリック
6. Googleアカウントを選択 → 「詳細」→「歩行年齢テスト（安全でないページ）に移動」→ 「許可」
7. **ウェブアプリのURL**（`https://script.google.com/macros/s/.../exec`）が表示される
8. このURLをコピーして、スタッフ・受診者に共有する

> **注意：** コードを修正した場合は「デプロイ → デプロイを管理 → ✏️編集 → バージョン:新しいバージョン → デプロイ」で更新が必要です。

## 7. データの読み込みとレポート作成

システムを開いたら、左側のパネルに情報を入力していきます。

### 7-1. 対象者情報を入力する

| 項目 | 入力内容 | 備考 |
|------|---------|------|
| 氏名 | 受診者の名前 | 例：山田 太郎 |
| 性別 | 男性 / 女性 | プルダウンで選択 |
| 実年齢 | 年齢（歳） | 計算に使用されるので正確に |
| 身長 | cm | レポート表示＋歩幅推定補助（倒立振子モデル `estimateLegLength()` での脚長算出）に使用 |
| 実施日 | 日付 | 初期値は本日の日付 |
| 管理番号 | 任意 | 施設の管理番号など |

### 7-2. phyphox データを読み込む

**方法①：CSVファイルをアップロード**

1. 「CSVを選択 / ドロップ / ペースト」ボタンをクリック
2. phyphox からエクスポートした CSV ファイルを選択
3. 「✓ 読み込み完了」と表示されたら成功

**方法②：ドラッグ＆ドロップ**

1. PC のファイルをブラウザ画面上にドラッグ
2. 青い点線の枠が表示されたらそのまま離す

**方法③：phyphox WiFi リモート取得（ローカルLAN専用）**

> ⚠️ **Netlify・GAS 公開 URL からは利用できません。** ローカル起動環境のみ対応（§3-3 参照）。

前提：
- phyphox の「Remote Access」が有効であること
- **ローカル起動の場合**：「Use HTTPS」が有効で、ブラウザで証明書を承認済みであること（§3-3 参照）

1. スマートフォンの phyphox で計測済みであること
2. 「IPアドレス」欄に phyphox の IP アドレスを入力（例：`192.168.1.5`）
3. 「取得」ボタンをクリック

**方法④：サンプルデータでテスト（動作確認用）**

1. 「サンプルデータでテスト」リンクをクリック
2. 架空のデータが自動入力され、レポートが表示される

### 7-3. 歩行距離を設定する

「測定値の調整」欄の上部に歩行距離の設定があります。

| 設定方法 | 操作 |
|---------|------|
| **プリセットボタン** | 「5m」「10m」「20m」ボタンをクリック（デフォルト: 10m） |
| **任意値入力** | 数値入力欄に 1〜100m の範囲で直接入力（0.5m 刻み） |

> **ポイント：** 計測時に歩いた距離に合わせて設定してください。歩行距離を変更すると、速度・歩幅が自動的に再計算されます。

### 7-4. 測定値を確認・調整する

CSV 読み込み後、自動計算された値が「測定値の調整」欄に表示されます。

- ストップウォッチで実際に計測した歩行速度がある場合 → **「歩くスピード」に入力**すると精度が向上します
- 値を変更すると、右のレポートがリアルタイムで更新されます

### 7-5. コメントを編集する（任意）

「総評・コメント」欄には、測定結果に基づいたコメントが**自動生成**されます。
内容を確認し、必要に応じて自由に編集できます。

### 7-6. バッチ処理（複数 CSV の一括解析）

複数の対象者の CSV ファイルを一括で解析する機能です。最大 50 件まで処理できます。

**手順**

1. 左パネルの計測データ取得セクションで **「📊 バッチ」タブ** を選択
2. （任意）**対象者情報 CSV** を選択 — 各ファイルに紐づく氏名・性別・年齢を指定できます
   - 列構成: `filename, userName, gender, age, height, userId`（1 行目はヘッダー必須）
   - **注意:** 対象者情報CSVは単純なカンマ区切りで解析しています。**フィールド値にカンマを含む場合や、引用符（ダブルクォート）で囲まれたフィールドには対応していません。** フィールド値にカンマを含めないでください。
3. **加速度 CSV** をクリックまたはドラッグ&ドロップで選択（複数選択可）
4. デフォルトの性別・年齢・歩行距離を設定
5. **「▶ 一括解析を実行」** ボタンをクリック
6. 解析結果がテーブルに表示される（速度・歩幅・テンポ・スコア・歩行年齢）

**結果の活用**

| ボタン | 機能 |
|--------|------|
| **📥 結果を CSV エクスポート** | 全結果を CSV ファイルとしてダウンロード |
| **📥 全員分を履歴 DB に保存** | 成功した全レコードを計測履歴データベースに保存 |

## 8. レポートの見方

レポートは表示設定によって最大 5 ページで構成されます。表示されるページは右上ボタンの ON/OFF 状態により変わります。

| ページ | 表示条件 |
|--------|---------|
| **Page 1**（受診者向けレポート） | 常に表示 |
| **Page 1.5**（推奨介入・運動アドバイス） | 「詳細アドバイス」ボタンが ON の場合のみ表示 |
| **Page 2**（地域在住者向け評価レポート） | 「地域在住向け」ボタンが ON の場合のみ表示 |
| **Page 3**（臨床向け詳細データ） | 「臨床向け」ボタンが ON の場合のみ表示 |
| **Page 4**（シューズレコメンド） | 「シューズレコメンド」ボタンが ON かつシューズデータ読込済みの場合のみ表示 |

### Page 1：受診者向けレポート

```
┌─────────────────────────────────────────┐
│ 歩行年齢テスト 結果レポート              │
│ 実施日: 2026.03.05  ID: 2026-W-1234     │
├─────────────────────────────────────────┤
│ 氏名: 山田 太郎 様  性別:男性  実年齢:65歳 │
├─────────┬───────────────────────────────┤
│         │  総合 歩行スコア               │
│  歩行年齢 │                              │
│   62 歳  │     74 点 / 100              │
│ 実年齢より│  ████████░░ 良好            │
│  3歳若い │                              │
```

### Page 2：地域在住者向け評価レポート

> **表示条件：** 画面右上の「**地域在住向け**」ボタンが ON の場合のみ表示されます。

地域PT・介護士・ケアマネージャー向けに、介護予防の観点で歩行能力を評価するページです。

**Section 1：転倒・生活機能リスク判定（信号機カード）**

転倒リスクと生活機能を3枚の信号機カード（🟢緑/🟡黄/🔴赤）で評価します。

| カード | 内容 | 判定基準 |
|-------|------|---------|
| 転倒リスク・地域活動 | 歩行速度ベース | 緑：≥ 1.0 m/s（低リスク） / 黄：0.8〜1.0 m/s（中リスク） / 赤：< 0.8 m/s（高リスク） |
| 余命・健康寿命 | 歩行速度ベース | 緑：≥ 1.0 m/s（平均以上） / 黄：0.8〜1.0 m/s / 赤：< 0.8 m/s（平均以下） |
| 歩幅 | 年齢・性別別Zスコア | 緑：≥−0.5SD（良好） / 黄：−1.5〜−0.5SD / 赤：<−1.5SD — 下肢筋力・歩幅訓練を推奨（Mobbs et al. Sensors 2025） |

**Section 2：フレイル・サルコペニアスクリーニング**

歩行速度・ケイデンス・歩行年齢からフレイルリスクを評価します。

**Section 3：歩行の質（生体力学指標）**

CSVデータが無い場合はカード自体は描画されますが、値は **N/A** と表示されます。

| 指標 | 正常値 | 意味 |
|------|--------|------|
| リズム安定性（CV） | < 3%: 安定 / 3〜5%: やや不安定 / ≥ 5%: 不安定 | 歩くリズムのばらつき（小さいほど安定）。なお Page 3「臨床向け」では < 4.0% を良好閾値として使用 |
| 左右対称性（SI） | < 5% | 左右の歩き方の差 |

> **CV（変動係数）の閾値はページ・用途により異なります。** Page 2（地域在住向け）は 3%/5% 基準（安定/やや不安定/不安定）、Page 3（臨床向け）は 4.0% を良好閾値として使用します。アドバイス自動選択では CV > 4.0% で `rhythmImpaired`（リズム異常）と判定し、地域在住向けアラートは CV ≥ 5.0% で追加表示されます。これは対象者・目的に応じた意図的な設計です。

**Section 4：総合アドバイス**

測定値に基づいた介入の優先順位と具体的アドバイスを表示します。

## 10. シューズレコメンド機能の使い方

歩行データに基づいて、最適なシューズを推薦する機能です。歩行速度・足の悩み・膝の状態・防水需要・着脱方法・使用目的・デザイン好み・性別・活動量を多次元で総合評価し、登録済みシューズをスコアリングします。

### 10-1. シューズレコメンドを ON にする

1. 画面右上の **「シューズレコメンド」ボタン** をクリック
2. ボタンが青くなり「ON」と表示される
3. 左パネルに「おすすめ靴 データベース」と「パーソナライズ設問」が表示される

### 10-2. パーソナライズ設問を設定する（Q1〜Q6）

左パネルの Q1〜Q6 の質問に回答することで、推薦精度が大幅に向上します。

| 質問 | 選択肢 | 推薦への影響 |
|------|--------|------------|
| **Q1. O脚・膝の悩み** | なし / 軽度 / 強い悩みあり | 膝悩みがある場合、安定性・クッション性の高い靴を優先。**「強い悩みあり」ではニーズアップ（KNEESUP）シリーズが強力に推奨** |
| **Q2. 使用シーン（防水）** | チェックボックス | **GORE-TEX素材 > 合皮・合成皮革 > 一般防水**の3段階で防水性能を評価 |
| **Q3. 足の悩み・特徴** | 特になし / 外反母趾 / 扁平足 / 外反母趾＋扁平足 / 幅広・甲高 | 足幅マッチング。**幅広・甲高選択時は 4E/5E 幅の靴に超幅広ボーナス** |
| **Q4. 靴の着脱方法** | こだわらない / 紐靴 / マジックテープ / サイドジッパー | **マジックテープ選択時はライフウォーカーのみ**に絞込。ジッパー・紐靴も対応モデルに限定 |
| **Q5. 主な使用目的** | 自動判定（推奨） / 日常のお出かけ / ウォーキング / アクティブ | 活動量レベルを手動で指定可能。**「自動判定」では歩行速度から自動で算出** |
| **Q6. デザインや着用シーン** | こだわらない / スポーティ・運動用 / キレイめ・お仕事用 | **スポーティ**: メッシュ・ニット素材を優先。**キレイめ**: 天然皮革・合皮素材を優先 |

> **活動量と着脱方法の自動連携：** 歩行速度が低い（< 0.8 m/s）場合、Q4 が「こだわらない」のときは自動でマジックテープ・ジッパー型を優先推薦します。

> **速度による自動絞込：** 歩行速度 ≤ 0.9 m/s の場合、ライフウォーカーシリーズのみが推薦されます（安全性を最優先するため）。

> **性別と商品モデルの自動連携：** 対象者情報の「性別」に基づき、男性にはメンズモデル、女性にはレディースモデルが自動的に優先されます（ユニセックス商品は性別を問わず推薦対象）。

### 10-3. シューズデータベースを準備する（管理者向け）

## 12. よくあるトラブルと解決方法

### Q1. 「データ行が少なすぎます」と表示される

**原因：** CSV ファイルのデータが短すぎる（歩行時間が短い）

**解決策：**
- 計測時間を 20 秒以上にして再計測する
- phyphox で「Linear Acceleration」を選択しているか確認する

---

### Q2. 「必要な列が見つかりません」と表示される

**原因：** CSV の列ヘッダーが正しくない（phyphox 以外のアプリのデータを使っている）

**解決策：**
- phyphox の **「Acceleration (without g)」**（重力補正あり加速度）でエクスポートする
- 1 行目が `Time (s), Linear Acceleration x (m/s^2), ...` であることを確認

---

### Q3. phyphox WiFi でデータを取得できない

**原因A：** 接続環境と HTTPS 設定が合っていない

**解決策：**
- **この機能はローカルLAN環境専用です。Netlify・GAS 公開 URL からは利用できません。** CSVエクスポート（方法A）をご利用ください。
- **ローカル起動（`file://` 等）の場合**：phyphox で「**Use HTTPS**」を有効にし、PC ブラウザでそのアドレス（例：`https://192.168.1.5`）を開いて証明書を承認してください。

**原因B：** ブラウザで phyphox の証明書を承認していない（ローカル起動時）

**解決策：** PCとスマホが同じWiFiに接続されているか確認してください。また、入力したIPアドレスが正しいこと（例: `192.168.1.5`）を確認してから再試行してください。

**原因C：** スマートフォンと PC が異なる WiFi に接続されている

**解決策：** 同じ WiFi ネットワーク（同じルーター・院内 LAN）に接続する

**原因D：** phyphox の Remote Access 自体が無効

**解決策：** phyphox の実験画面 →「⋮」→「Remote Access」をタップして有効にする

> ℹ️ phyphox WiFi リモート取得はローカルLAN環境専用です。ローカル起動（file:// 等）の HTTPS 環境では phyphox の「Use HTTPS」が必要です（§3-3 参照）。Netlify・GAS 公開 URL からは利用できません。

### Q4. PDF の生成に失敗する / 真っ白になる

**原因：** ブラウザの設定またはメモリ不足

**解決策：**
- Chrome の最新版を使用する
- ブラウザのタブを閉じてメモリを解放してから再試行
- シューズレコメンド（Page 4）をオフにしてから PDF 出力を試す

---

### Q5. シューズ画像が表示されない

**原因：** ローカル（file://）環境では CORS 制限により画像を取得できない

**解決策：** GAS または Netlify にデプロイした環境で使用する

---

### Q6. GAS デプロイ後に「スクリプトエラー」が出る

**原因：** Code.gs または index.html のコードが古いバージョンのまま

**解決策：**
1. Apps Script の編集画面を開く
2. Code.gs と index.html を最新のファイルで上書きする
3. **「デプロイ → デプロイを管理 → ✏️編集 → バージョン:新しいバージョン → デプロイ」** を実行する

---

### Q7. 歩行年齢や速度の計算値がおかしい

**原因：** スマートフォンを手に持って歩いた / 計測時間が短すぎる

**解決策：**
- スマートフォンを腰にしっかり固定して再計測する
- 「測定値の調整」欄でストップウォッチ計測の速度を手動入力する
