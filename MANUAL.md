## 1. このシステムでできること

スマートフォンで歩き方を計測して、**「歩行年齢」** を診断するシステムです。

```
スマートフォンで歩く
       ↓
データを自動解析
       ↓
歩行年齢・スコアがわかる
       ↓
A4 レポートを PDF で出力・共有
```

**診断できる内容**

| 項目 | 説明 |
|------|------|
| 歩行年齢 | 「何歳相当の歩き方か」を推定 |
| 総合スコア | 0〜100 点で評価 |
| 歩くスピード | m/s（毎秒何メートル進むか） |
| 歩幅の広さ | 1 歩の長さ（m） |
| 歩くテンポ | 1 分間の歩数（歩/分） |
| リズムの安定性 | 一定のリズムで歩けているか |
| 左右対称性 | 左右の歩き方のバランス |

## 2. 必要なものを用意する

### 2-1. 機器・アプリ

| 必要なもの | 詳細 |
|-----------|------|
| **スマートフォン** | iPhone または Android（センサー内蔵のもの） |
| **phyphox アプリ** | 無料（次のセクションで説明） |
| **PC またはタブレット** | Chrome / Safari / Edge など最新ブラウザ推奨 |

### 2-2. 計測環境

- **歩く場所：** 直線で **10 メートル以上** 歩けるスペース
- **床の状態：** 平坦な床（屋内廊下・体育館など）
- **靴：** 普段履いている靴でOK
- **補助具：** 杖・歩行器を使っている方はそのまま計測可能

> **ポイント：** 10 m のラインをテープなどで床に貼っておくと計測が楽になります。

## 3. phyphox アプリのセットアップ

phyphox（フィフォックス）は、スマートフォンの加速度センサーを使って計測するアプリです。**無料**で使えます。

### 3-1. インストール

| 端末 | 入手先 |
|------|--------|
| iPhone (iOS) | App Store で「phyphox」を検索 |
| Android | Google Play で「phyphox」を検索 |

> 開発元：RWTH Aachen University（ドイツのアーヘン工科大学）

### 3-2. 計測メニューの設定

1. phyphox を起動する
2. 画面下の **「+」ボタン** をタップ
3. **「Acceleration (without g)」**（重力を除いた加速度）を選ぶ

   > 日本語表示の場合は **「加速度（重力補正）」** または **「Linear Acceleration」**

4. 新しい実験が作成される

### 3-3. WiFi リモート機能の設定（任意・便利機能）

計測中にスマートフォンを操作しなくて済む方法です。スマートフォンと PC が同じ WiFi に接続されている場合に使えます。

1. phyphox の実験画面右上の **「⋮」（縦三点）→「Remote Access」** をタップ
2. **「Port 8080」** が表示され、同じ WiFi 内の PC から接続できるようになる
3. 表示された **IP アドレス**（例：`192.168.1.5`）をメモしておく

## 4. 歩行計測の手順

### 4-1. スマートフォンの装着位置

腰のベルトや**ズボンの腰部分**（腰の中央・背骨の真後ろ付近）に固定するのが最適です。

```
      [頭]
       │
      [肩]
       │
  ━━━━━━━━  ← 腰（ここにスマートフォンを固定）
       │
      [足]
```

**固定方法（どれか 1 つ）**
- ポケットに入れる（腰ポケット）
- ウエストポーチに入れて腰に巻く
- スマートフォンホルダーをベルトに装着
- スポーツアームバンドを腰に巻いて装着

> **注意：** 手に持って歩くと正確に計測できません。必ず腰に固定してください。

### 4-2. 計測手順（ステップバイステップ）

```
① 準備する（10m ライン）
   ↓
② phyphox で計測スタート
   ↓
③ 3〜5 秒 その場で静止（準備のため）
   ↓
④ 普通のペースで 10m 歩く
   ↓
⑤ 10m 歩いたら折り返す（または止まる）
   ↓
⑥ ①〜⑤ を 2〜3 往復繰り返す（計 20〜30 秒以上）
   ↓
⑦ phyphox で計測ストップ
   ↓
⑧ データをエクスポート（次ページ参照）
```

> **ポイント：** 「テスト中」と意識しすぎず、**いつも通りのペース**で歩いてください。

### 4-3. phyphox からデータをエクスポートする

**方法A：CSVファイルに保存する**

1. 計測停止後、phyphox 右上の **「⋮」→「Export Data」** をタップ
2. **「CSV（Comma Separated Values）」** を選択
3. メールや AirDrop などで PC に送る

**方法B：WiFi リモートでPCから直接取得する（推奨）**

1. phyphox で計測開始・歩行・計測停止まで完了させる
2. 診断システムの左パネル「phyphox WiFi リモート取得」欄に IP アドレスを入力
3. 「取得」ボタンを押すと自動でデータが読み込まれる

## 6. Google Apps Script（GAS）でのセットアップ

GAS を使うと、URL をスタッフで共有するだけで複数人が使える環境を作れます。

### 6-1. Google アカウントの準備

Google アカウント（Gmail アカウント）が必要です。持っていない場合は [accounts.google.com](https://accounts.google.com) で作成してください。

### 6-2. Apps Script プロジェクトを作成する

1. ブラウザで [script.google.com](https://script.google.com) を開く
2. 左上の **「新しいプロジェクト」** をクリック
3. プロジェクト名を「歩行年齢テスト」などに変更（左上の「無題のプロジェクト」をクリックして入力）

### 6-3. Code.gs を設定する

1. 画面左に **「Code.gs」** が表示されている（デフォルトで作成済み）
2. `Code.gs` をクリックして開く
3. 最初から書かれているコードを **すべて削除**
4. プロジェクトの `Code.gs` ファイルの内容を**丸ごとコピー＆ペースト**
5. 上部の **「💾 保存」ボタン**（または Ctrl+S / Cmd+S）で保存

```
【Code.gs に貼り付ける内容のイメージ】

function doGet(e) { ... }         ← HTMLページを配信する関数
function fetchCsvData(url) { ... } ← シートデータを取得する関数
function fetchImageAsBase64(url) { ... } ← 画像を取得する関数
```

### 6-4. index.html を作成する

1. 画面左の **「＋」ボタン → 「HTML」** をクリック
2. ファイル名を **「index」**（拡張子なし）と入力して Enter
3. `index.html` ファイルが作成される
4. プロジェクトの **`index.html`** の内容を**丸ごとコピー＆ペースト**
5. 保存（Ctrl+S / Cmd+S）

### 6-5. ウェブアプリとしてデプロイする

1. 画面右上の **「デプロイ」→「新しいデプロイ」** をクリック
2. 「種類を選択」右の**歯車アイコン** をクリック → **「ウェブアプリ」** を選択
3. 以下のように設定する

   | 項目 | 設定値 |
   |------|--------|
   | 説明（任意） | 歩行年齢テスト v6 |
   | 次のユーザーとして実行 | **自分** |
   | アクセスできるユーザー | **全員** |

4. **「デプロイ」** ボタンをクリック
5. 初回は「アクセスを承認」の画面が出る → **「アクセスを承認」** をクリック
6. Googleアカウントを選択 → 「詳細」→「歩行年齢テスト（安全でないページ）に移動」→ 「許可」
7. **ウェブアプリのURL**（`https://script.google.com/macros/s/.../exec`）が表示される
8. このURLをコピーして、スタッフ・受診者に共有する

> **注意：** コードを修正した場合は「デプロイ → デプロイを管理 → ✏️編集 → バージョン:新しいバージョン → デプロイ」で更新が必要です。

## 7. データの読み込みとレポート作成

システムを開いたら、左側のパネルに情報を入力していきます。

### 7-1. 対象者情報を入力する

| 項目 | 入力内容 | 備考 |
|------|---------|------|
| 氏名 | 受診者の名前 | 例：山田 太郎 |
| 性別 | 男性 / 女性 | プルダウンで選択 |
| 実年齢 | 年齢（歳） | 計算に使用されるので正確に |
| 身長 | cm | レポート表示のみ |
| 実施日 | 日付 | 初期値は本日の日付 |
| 管理番号 | 任意 | 施設の管理番号など |

### 7-2. phyphox データを読み込む

**方法①：CSVファイルをアップロード**

1. 「CSVを選択 / ドロップ / ペースト」ボタンをクリック
2. phyphox からエクスポートした CSV ファイルを選択
3. 「✓ 読み込み完了」と表示されたら成功

**方法②：ドラッグ＆ドロップ**

1. PC のファイルをブラウザ画面上にドラッグ
2. 青い点線の枠が表示されたらそのまま離す

**方法③：phyphox WiFi リモート取得**

1. スマートフォンの phyphox で計測済みであること
2. 「IPアドレス」欄に phyphox の IP アドレスを入力（例：`192.168.1.5`）
3. 「取得」ボタンをクリック

**方法④：サンプルデータでテスト（動作確認用）**

1. 「サンプルデータでテスト」リンクをクリック
2. 架空のデータが自動入力され、レポートが表示される

### 7-3. 測定値を確認・調整する

CSV 読み込み後、自動計算された値が「測定値の調整」欄に表示されます。

- ストップウォッチで実際に計測した歩行速度がある場合 → **「歩くスピード」に入力**すると精度が向上します
- 値を変更すると、右のレポートがリアルタイムで更新されます

### 7-4. コメントを編集する（任意）

「総評・コメント」欄には、測定結果に基づいたコメントが**自動生成**されます。
内容を確認し、必要に応じて自由に編集できます。

## 8. レポートの見方

レポートは A4 サイズ 2〜3 ページで構成されています。

### Page 1：受診者向けレポート

```
┌─────────────────────────────────────────┐
│ 歩行年齢テスト 結果レポート              │
│ 実施日: 2026.03.05  ID: 2026-W-1234     │
├─────────────────────────────────────────┤
│ 氏名: 山田 太郎 様  性別:男性  実年齢:65歳 │
├─────────┬───────────────────────────────┤
│         │  総合 歩行スコア               │
│  歩行年齢 │                              │
│   62 歳  │     74 点 / 100              │
│ 実年齢より│  ████████░░ 良好            │
│  3歳若い │                              │
├─────────┴───────────────────────────────┤
│ レーダーチャート │ 測定値・スコア・判定    │
│  スピード       │ スピード  1.18m/s  74点 │
│  歩幅    テンポ │ 歩幅     0.65m    71点 │
│  リズム安定     │ テンポ   108歩/分  68点 │
│  左右対称       │                        │
├─────────────────────────────────────────┤
│ 担当者コメント（自動生成・編集可）         │
└─────────────────────────────────────────┘
```

**歩行年齢の読み方**

| 表示 | 意味 |
|------|------|
| 実年齢より〇歳**若い** | 同年代より良好な歩き方 |
| 実年齢と**ほぼ同じ** | 年齢相応の歩き方 |
| 実年齢より〇歳**高い** | 歩行能力の低下傾向あり |

**スコアの判定目安**

| スコア | 判定 |
|--------|------|
| 80〜100 点 | 優秀（同年代の上位 16%） |
| 60〜79 点 | 良好（同年代の上位 30%） |
| 40〜59 点 | 標準（平均的） |
| 20〜39 点 | やや低下（同年代の下位 30%） |
| 0〜19 点 | 低下（同年代の下位 16%） |

---

### Page 2：専門家向け詳細データ

医療・介護専門職向けの詳細データです。

**Section 1：臨床リスクスクリーニング**

歩行速度を 4 つの臨床閾値で評価します。

| 閾値 | 意味 |
|------|------|
| 0.8 m/s 未満 | 転倒・入院リスクが有意に上昇 |
| 1.0 m/s 未満 | サルコペニア・フレイルの精査推奨 |
| 1.0 m/s 未満 | 横断歩道の安全通過が困難な可能性 |
| 1.2 m/s 未満 | 高機能群（良好な生命予後水準）に未達 |

**Section 2：歩行の質的評価（Biomechanics）**

CSV データを読み込んだ場合のみ表示されます。

| 指標 | 正常値 | 意味 |
|------|--------|------|
| リズム安定性（CV） | < 4.0% | 歩くリズムのばらつき |
| 左右対称性（SI） | < 5.0% | 左右の歩き方の差 |
| 歩行制御効率（Walk Ratio） | 0.38〜0.42 m/Hz | 中枢神経の歩行制御効率 |
| 体幹動揺強度（RMS） | 0.50〜2.50 m/s² | 体幹の揺れの大きさ |

**Section 3：同年代比較（Z スコア表）**

年齢・性別で同年代の平均値（μ）と標準偏差（σ）を比較します。

**Section 4：歩行周期波形**

加速度センサーの波形グラフです。
- **青いピーク：** 解析に使用した定常区間のステップ
- **薄青色の帯：** 定常区間（加速・減速を除いた中央部分）

## 9. シューズレコメンド機能の使い方

歩行データに基づいて、最適なシューズを推薦する機能です。

### 9-1. シューズレコメンドを ON にする

1. 画面右上の **「シューズレコメンド」ボタン** をクリック
2. ボタンが青くなり「ON」と表示される
3. 左パネルに「おすすめ靴 データベース」が表示される

### 9-2. シューズデータベースを準備する（管理者向け）

Google スプレッドシートでシューズデータを管理します。

**スプレッドシートの列構成（1 行目はヘッダー）**

| 列名 | 内容 | 例 |
|------|------|-----|
| `name` | 商品名 | ゲルライドウォーク 2 |
| `url` | 商品ページ URL | https://walking.asics.com/... |
| `image_url` | 商品画像 URL | https://images.asics.com/... |
| `price` | 価格 | ¥17600 |
| `features` | 特徴（`|` で区切る） | ロッカー\|推進\|クッション\|安定性 |
| `description` | 商品説明 | 省エネ設計の... |

**スプレッドシートを公開する手順**

1. Google スプレッドシートを開く
2. 上部メニュー **「ファイル」→「共有」→「ウェブに公開」** をクリック
3. 「リンク」タブを開き、形式を **「カンマ区切り（.csv）」** に変更
4. **「公開」** ボタンをクリック
5. 表示された URL をコピーする

   > 例：`https://docs.google.com/spreadsheets/d/XXXXXX/pub?output=csv`

### 9-3. シューズデータを読み込む

1. 左パネル「おすすめ靴 データベース」欄の URL 入力欄にコピーした URL を貼り付け
2. **「読込」** ボタンをクリック
3. 「✓ ○件の靴データを読み込みました」と表示されたら成功
4. Page 3（シューズレコメンド）が自動的に表示される

### 9-4. レコメンド結果の見方

歩行データから必要な靴の特性が自動計算され、登録済みシューズをスコアリング。上位 3 件が表示されます。

```

│ 歩行プロファイル                          │
│ 歩行速度 0.95m/s[注意] 歩幅 0.60m 推進 3E幅 │
├──────────────────────────────────────────┤
│ 🥇 1st  ゲルライドウォーク 2           │
│    ¥17,600                              │
│    [ロッカー][推進][クッション][安定性]  │
│    適合度 ████████░░ 80%               │
│    ✓ 安定性・ウォーキングサポート        │
│    ✓ クッション性                       │
│    [QRコード] ← スマートフォンで読取可   │
└──────────────────────────────────────────┘
```

## 10. PDF出力・印刷・共有

### 10-1. PDF として保存する（「PDF出力」ボタン）

1. 画面右上の青い **「PDF出力」** ボタンをクリック
2. 自動でページが生成される（10〜30 秒かかります）
3. `歩行年齢テスト_山田太郎_2026.03.05.pdf` という名前でダウンロードされる

> **出力されるページ：**
> - Page 1（受診者向け）は常に出力
> - Page 2（専門家向け）は常に出力
> - Page 3（シューズレコメンド）はシューズレコメンドが ON のときのみ出力

### 10-2. 印刷する（「印刷」ボタン）

1. **「印刷」** ボタンをクリック
2. ブラウザの印刷ダイアログが開く
3. 用紙：**A4**、余白：**なし（最小）**、印刷範囲：**全て** に設定

### 10-3. 共有する（「共有」ボタン）

1. **「共有」** ボタンをクリック
2. PDF が自動生成され、OS の共有メニューが開く

   | 端末・環境 | 動作 |
   |-----------|------|
   | iPhone / iPad（iOS 15+） | AirDrop・メール・LINE 等でPDFを共有 |
   | Android（Chrome） | Nearby Share・メール等でPDFを共有 |
   | PC（デスクトップ） | PDFをダウンロード |

## 11. よくあるトラブルと解決方法

### Q1. 「データ行が少なすぎます」と表示される

**原因：** CSV ファイルのデータが短すぎる（歩行時間が短い）

**解決策：**
- 計測時間を 20 秒以上にして再計測する
- phyphox で「Linear Acceleration」を選択しているか確認する

---

### Q2. 「必要な列が見つかりません」と表示される

**原因：** CSV の列ヘッダーが正しくない（phyphox 以外のアプリのデータを使っている）

**解決策：**
- phyphox の **「Acceleration (without g)」**（重力補正あり加速度）でエクスポートする
- 1 行目が `Time (s), Linear Acceleration x (m/s^2), ...` であることを確認

---

### Q3. phyphox WiFi でデータを取得できない

**原因A：** スマートフォンと PC が異なる WiFi に接続されている

**解決策：** 同じ WiFi ネットワーク（同じルーター）に接続する

**原因B：** GAS / Netlify 環境（HTTPS）で HTTP に直接アクセスできない

**解決策：** Netlify 環境では自動的にサーバー経由で取得されます。GAS 環境ではこの機能は使えないため、CSVファイルをエクスポートしてアップロードしてください。

**原因C：** phyphox の Remote Access が有効になっていない

**解決策：** phyphox の実験画面 →「⋮」→「Remote Access」を有効にする

---

### Q4. PDF の生成に失敗する / 真っ白になる

**原因：** ブラウザの設定またはメモリ不足

**解決策：**
- Chrome の最新版を使用する
- ブラウザのタブを閉じてメモリを解放してから再試行
- 画像の多い Page 3（シューズレコメンド）をオフにしてから PDF 出力を試す

---

### Q5. シューズ画像が表示されない

**原因：** ローカル（file://）環境では CORS 制限により画像を取得できない

**解決策：** GAS または Netlify にデプロイした環境で使用する

---

### Q6. GAS デプロイ後に「スクリプトエラー」が出る

**原因：** Code.gs または index.html のコードが古いバージョンのまま

**解決策：**
1. Apps Script の編集画面を開く
2. Code.gs と index.html を最新のファイルで上書きする
3. **「デプロイ → デプロイを管理 → ✏️編集 → バージョン:新しいバージョン → デプロイ」** を実行する

---

### Q7. 歩行年齢や速度の計算値がおかしい

**原因：** スマートフォンを手に持って歩いた / 計測時間が短すぎる

**解決策：**
- スマートフォンを腰にしっかり固定して再計測する
- 「測定値の調整」欄でストップウォッチ計測の速度を手動入力する
